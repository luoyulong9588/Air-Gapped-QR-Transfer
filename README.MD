# 🚀 Air-Gapped QR Transfer (V4 Pro)

**基于二维码的断网文件传输系统**

这是一个利用高密度二维码感官通道（Optical Channel）实现物理隔离传输的工具。通过浏览器即可在电脑与手机、或两台手机之间快速传递文件，无需 Wi-Fi、蓝牙或数据线。
（因为医院内网行为管理软件会禁止使用U盘或上报USB接入设备，故设计这个隔空文件传输工具。基于HID模拟的回传工具正在开发中）
---

## ✨ 核心特性

* **⚡ 极速压缩 (Pako/Zlib)**：集成 `pako.js` 压缩库。在发送前自动对文件进行 GZIP 级压缩，对于文本、代码等文件可减少 **70% - 90%** 的传输量。
* **🛡️ 混合动力容错 (Sequential + Fountain Code)**：
    * **顺序模式**：优先按顺序发送分片，让接收端快速建立文件结构。
    * **喷泉模式**：在顺序循环后自动切换到卢比变换（LT Codes）逻辑，生成无限补丁包。接收端只需捕获足够的任意分片即可完成消元解码，无惧摄像头丢帧。
* **🔄 智能连续传输**：
    * **自动重置**：发送端切换文件时，接收端通过 `K值校验` 自动感知并重置解码器，无需手动刷新页面。
    * **内存优化**：自动释放 `Blob URL`，支持连续传输多个文件而不卡顿。
* **📱 广角大框扫描**：优化了浏览器的摄像头调用策略，利用 **85% 屏幕面积** 作为有效感官区。
* **📄 零乱码保障**：通过元数据（Metadata）精确记录原始文件字节长度，彻底解决由于分片对齐导致的末尾乱码问题。

## 📖 使用指南

### 1. 准备环境
由于浏览器摄像头权限限制，**接收端（扫码端）必须通过以下方式访问**：
* 使用 **HTTPS** 协议部署。
* 或者在本地使用 `localhost` 访问。

### 2. 发送文件 (Sender)
1.  在电脑端打开 `hybrid-sender.html`。
2.  点击 **“选择文件”**，系统会自动完成压缩并显示文件详情。
3.  点击 **“开始传输”**，屏幕将开始循环显示二维码。

### 3. 接收文件 (Receiver)
1.  在手机端打开 `hybrid-receiver.html`。
2.  允许摄像头权限，并将扫描框对准发送端的二维码。
3.  观察进度条，当显示 “✅ 解压还原成功” 时，点击 **“保存并解压文件”**。

---

## ⚠️ 注意事项

1.  **光线与对焦**：压缩后的二维码较为精细，请确保环境光线充足，并保持手机摄像头与屏幕距离约 15-25 厘米。
2.  **文件大小**：建议传输 **10MB 以内** 的文件体验最佳。
3.  **浏览器兼容性**：建议使用最新版 Chrome、Edge 或 Safari (iOS)。
